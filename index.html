About Me

I am a graduate student at Northeastern University, focusing on MLE. I enjoy building recommendation systems and fine-tuning large language models.

I chose to study Web technologies not just to build web pages, but to better understand the overall workflow of recommendation systems. From front end to back end, the web is the first gateway through which users interact with a recommender. By learning HTML / CSS / JavaScript, I can design more effective user interaction logging, which provides higher-quality data for recommendation algorithms.

For recommendation systems, data is the core. Understanding how data flows are collected, transmitted, and presented helps me connect user interaction logs with the distributed computing model. For example, user clicks, browsing, and search behaviors can be broken into small data units processed in the Map phase, then aggregated in the Reduce phase into features that drive training and ranking.

Learning Web development allows me to connect algorithms and engineering more tightly: the front end captures user behavior and displays personalized results, while the back end and distributed systems handle large-scale data processing. This forms a closed loop that not only improves recommendation performance but also deepens my understanding of system architecture as a whole.
